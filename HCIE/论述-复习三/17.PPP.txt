问题一：
PAP 是通过二次握手身份验证协议，PAP 被认证方通过 Authentication-Request 报文以明文方式传输用户名和密码，安全性较低。
CHAP 认证是三次握手身份验证协议，CHAP 被认证方将认证 ID、随机数、认证密码进行 HASH 计算，通过 Response 报文传递 HASH 密文，
另外认证方通过挑战消息提供的随机数可以防止重放攻击，相对而言，CHAP 认证更为安全。

问题二：
PPPoE 也就是 PPP Over Ethernet，该技术同时结合了以太网和 PPP 的诸多优点，具体介绍如下：
1、PPPoE 继承了以太网接入便捷、经济的优先，目前广泛使用的家庭宽带接入就使用了 PPPoE 协议。
2、PPPoE 包含了 PPP 的强大功能，所有能被 PPP 封装的协议都可以被 PPPoE 进行传输。
3、PPPoE 可以使用 PPP 的 PAP 或者 CHAP 认证，实现安全接入，可以通过部署认证服务器实现统一认证管理。
4、PPPoE 通过 NCP 可以为用户提供动态 IP 地址分配模式，无需任何配置，就可进行简单的网管维护，同时可以根据分配的 IP 地址很好的定位用户在网络中的活动。
5、PPPoE 通过唯一的 Session-id 区分用户流量，可以方便的查看用户在线、离线状态，从运营商角度来看，计费方式灵活方便。

问题三：
1、可以建立邻居关系
OSPF 通过 Hello 报文进行邻居关系的建立与维系，在 P2P 网络类型中，Hello 报文的源地址为物理接口地址，目标地址为 224.0.0.5。
邻居关系可以正常建立，原因如下：
（1）PPP 通过 NCP(IPCP)协议可以通告对端自身 /32 的主机路由，不同网段的接口 IP 也可以完成 PPP 二层封装实现可达。
（2）OSPF 在 P2P 网络类型中默认不检查报文源地址是否与接收接口地址在同一网段。
（3）接口启用 OSPF 之后可以处理去往 224.0.0.5 的 Hello 报文并检查相应参数（P2P 网络类型中忽略子网掩码检查），参数一致则可以建立邻居关系。

2、可以计算路由
OSPF 主要通过 LSA 进行拓扑计算和路由计算，在 P2P 网络类型中报文可以正常交互，因此 LSDB 链路状态数据库可以正常同步。
在 P2P 网络类型中，拓扑信息和路由信息都通过 LSA-1 Router LSA 携带，路由可以正确计算，原因分析如下：
（1）拓扑计算
路由器自身产生的 LSA-1 Router LSA，通过 Link Type P2P 来描述拓扑信息，具体字段如下：
Link-Type：P2P
Link-ID：邻居设备的 Router-id
Data：自身接口 IP
Metric：接口链路开销
这样路由器以自身为根可以正确计算去往邻居设备的拓扑，完成拓扑计算。

（2）路由计算
路由器自身产生的 LSA-1 Router LSA，通过 Link Type Stubnet 来描述路由信息，具体字段如下：
Link-Type：Stubnet
Link-ID：网络号
Data：子网掩码
Metric：接口链路开销
完成上述拓扑计算之后，路由器将 Stubnet 中的路由信息添加到当前 SPT 最短路径树上，完成路由计算。

综上，在 P2P 网络类型中，接口 ip 不再同一网段，OSPF 可以正常建立邻居关系和路由计算。
