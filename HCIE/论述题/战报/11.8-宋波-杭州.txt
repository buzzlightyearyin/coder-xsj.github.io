            第二小问：PPPPOE的优势
  
              
            答： 1、 PPPOE网络搭建简便便捷，目前广泛使用的的家庭宽带接入就使用了pppoe协议。
                 
                 2、 PPPoE包含了PPP的强大功能，任何能被PPP封装的协议都可以通过PPPoE传输

第二题:EBGP排障：原题。


 解答如下：
       第一问：如何排查问题，写出必要的排障使用的命令

    答：一：如果AR2和AR4是通过互联接口建立邻居关系，可以根据以下步骤进行排查：

         1.在AR2和AR4上使用ping对端接口地址，来检测直连的联通性，然后AR2和AR4的互联接口是否在同一网段，使用dispaly interface 来查看，

         2.检查AR2和AR4的邻居状态，使用命令 dis bgp peer，来查看，根据不同的状态来进行处理:

           A.邻居关系停留在idle状态，在该场景中则可能由于：AS号配置错误、router-id冲突，或配置了peer ignore 等原因造成；

           B.若邻居关系一直停留在no neg 状态，表示BGP连接的能力没有协商，一般是afi配置错误
 
           C.邻居关系停留在connect状态，在该场景中则可能由于：R2或R4配置了认证，对端没有配置或两者认证信息不一致。
         
         3.也可以使用dispaly bgp error 根据错误信息，进行定位查看来解决,可以进入BGP下，使用display this，检查配置问题，逐步排查；

         4.检查两台设备上是否存在相应的流量过滤行为

          A.可能过滤了179端口，通过检查display tcp  status 来查看是否开放了179端口；

          B.可能过滤了相应的路由，通过dispaly acl all、 dispaly ip ip-prefix 等命令来查看，如存在相应的路由过滤，放行修复。
        
          

        二：如果AR2和AR4是通过环回接口建立邻居关系，可以根据以下步骤进行排查：

         1.在设备上ping对端设备的环回接口地址看是否可达，若不可达则：

             A.如果环回口使用静态路由可达，那么先检查互联的接口地址是否在同一个网段，通过dis play interface和ping命令来检查，然后再检查静态路由配置是否正确

             B.如果环回是通过动态路由可达，通过ping和dispalay ip routing命令 查看是否有对端的环回接口地址，然后再检查底层的动态路由协议是否配置正确 

          2.检查AR2和AR4的邻居状态，使用命令 dis bgp peer，来查看，根据不同的状态来进行处理

             A.邻居关系停留在idle状态，在该场景中则可能由于：更新源地址配置错误、AS号配置错误、router-id冲突,未配置EBGP多跳，配置了peer ignore等原因造成；

             B.若邻居关系一直停留在no neg 状态，表示BGP连接的能力没有协商，一般是afi配置错误

             C.邻居关系停留在connect状态，在该场景中则可能由于：R2或R4配置了认证，对端未配置或认证信息不一致；

             D.邻居关系停留在Active状态，在该场景中则可能由于使用环回口建立EBGP对等体但是没有修改更新源。

         3.也可以使用dispaly bgp error 根据错误信息，进行定位查看来解决；可以进入BGP下，使用display this，检查配置问题，逐步排除；
         
         4.检查两台设备上是否存在相应的流量过滤行为

            A.可能过滤了179端口，通过检查display tcp  status 来查看是否开放了179端口

            B.可能过滤了相应的路由，通过dispaly acl all、dispaly ip ip-prefix 等命令来查看，如存在相应的路由过滤，放行修复。
        

          

二：如果你是ISP1的管理员，ISP1和ISP2是不同的运行商，在网路没问题的时候，如何控制让10.1.1.0/24 选择AR1-AR3的路径，请写出2个方案

     答：根据题本拓扑以及相关信息可以部署以下两个方案来实现：

        1.在AR2上针对peer AR4的出方向配置策略，增加路由10.1.1.0/24的MED值，使得传递给AR4的关于10.1.1.0/24的路由MED值较大，从而AS200选择MED值较小的路径。

        具体配置命令如下：

        A.使用acl将10.1.1.0/24的路由匹配出来

        acl 2000
        rule permit source 10.1.1.0 0.0.0.255

        B.配置route-policy 并调用ACL

        route-policy MED permit node 10
        if-match acl 2000
        apply cost 100
        route-policy MED permit node 20

        C.进入BGP，在指向R4的出方向调用路由策略

        bgp 100
        peer X.X.X.X route-policy MED export      // X.X.X.X 为R4与R2建立EBGP邻居使用的IP地址

       2.在AR2上针对peer AR4 的出方向配置策略，增加10.1.1.0/24路由的AS-PATH值，使得此路由传递给AR4时候，as-path数量变多，从而是得AS200选择as-path数量较少的路径。

       具体配置命令如下：

       A.使用ACL 将10.1.1.1的路由匹配出来

       acl 2000
       rule permit source 10.1.1.0 0.0.0.255
    
       B.配置route-policy 调用acl 2000

       route-polcy AS  permit node 10
       if-match acl 2000
       apply as-path 100 additive  
       route-policy as-path permit node 20
 
       C.进入bgp，在指向R4的出方向上调用路由策略

       bgp 100
       peer X.X.X.X route-policy AS export        // X.X.X.X 为AR4与AR2建立EBGP邻居使用的IP地址
